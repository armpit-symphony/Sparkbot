Sparkbot v1.6.54

Focus: Agents and Controls cleanup, Workstation Specialty Wing agent seating, packaged specialist agents, and downloader alignment.

- Removed the obsolete Active custom agents display from Controls while preserving custom agent creation, persistence, and backend API support.
- Moved Spawn Agent to the top of Controls > Agents above Model Overrides for a clearer creation-first workflow on desktop and mobile.
- Added packaged built-in agents: Meetings Manager, Web Designer, Marketing Agent, and Business Analyst.
- Fresh installs and upgraded installs receive the new packaged agents from the existing backend registry; user-created custom agents remain preserved.
- Specialty Wing office cards now have dropdowns backed by the shared Controls Agents list, defaulting to @meetings_manager, @researcher, @analyst, @writer, and @workstation_backup_1 with local persistence for office assignments.
- Specialty Wing office side panels now include a model selector that saves through the same per-agent model override system used by Controls.
- New Workstation meeting setup preloads @meetings_manager into the first seat while leaving the seat editable.
- Promoted Spine Ops to the operator-facing Command Center while preserving the existing /spine route and adding /command-center as an alias.
- Moved/surfaced Room Persona, System Health, Computer Control, Token Guardian, and Task Guardian in Command Center, with Room Persona at the top.
- Command Center now uses the same top tab/header flow as the main Sparkbot pages instead of the separate left-side layout.
- Controls is now more configuration-focused, with a Command Center shortcut and operational panels removed from the visible Controls flow.
- Planned Command Center actions without complete backend context are disabled or clearly marked as not configured/read-only instead of appearing as live controls.
- Advanced backend, frontend, Tauri shell, README, public downloader, capabilities docs, release notes, service worker, and packaging notes to v1.6.54.
